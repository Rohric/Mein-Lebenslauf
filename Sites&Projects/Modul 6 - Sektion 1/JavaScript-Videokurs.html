1 · Variablen

Hier geht es um Variablen.
🔹 Merksatz für let (Variable, veränderbar)
„Eine Variable ist wie ein Schließfach mit Nummer:
Die Nummer (Name) bleibt gleich,
aber der Inhalt (Wert) kann jederzeit gewechselt werden.“

🔹 Merksatz für const (Konstante, unveränderbar)
„Eine Konstante ist wie ein versiegeltes Schließfach:
Die Nummer (Name) bleibt gleich,
und der Inhalt (Wert) darf nicht mehr verändert werden.“

Man unterscheidet dabei zwischen
let & const
"var" war gestern!

Beispiel

let meineVariable= 27; // let kann verändert werden.
console.log(meineVariable);
meineVariable = 32;
console.log(meineVariable)

const meineVariable2= 27; // const kann nicht verändert werden, kann auch zu fehlermeldungen führen.
console.log(meineVariable2);
meineVariable = 32;
console.log(meineVariable2)

3. Strings und Numbers

Was sind eigentlich.. Strings & Numbers?

  • Mit Strings kann man verbinden, zählen, verändern
  • Numbers = Zahlen (egal ob Ganzzahl oder Kommazahl)
  • Mit Zahlen kann man rechnen
  • Achtung: Strings und Numbers können sich vermischen
  • + kann beides verbinden → ergibt dann einen String
String Number Combination Not a Number

Strings vs Numbers (Beispiele)

Kategorie String (Text) Number (Zahl)
Definition Texte in " ", ' ' oder ` ` Zahlen (Ganzzahl oder Kommazahl)
Beispiel "Hallo" 42 oder 3.14
Operator + Verbindet Texte: "Hallo " + "Welt" → "Hallo Welt" Rechnet: 10 + 5 → 15
Länge / Größe .length gibt Zeichenanzahl zurück Keine .length, nur Zahlenwert
Methoden .toUpperCase(), .toLowerCase(), .includes('...') Mathe-Operatoren: + - * / %
Kombination "Ich bin " + 28 + " Jahre" → "Ich bin 28 Jahre" 10 * 2 → 20

Hinweis: Mischst du String + Zahl mit +, wird meist ein String (z. B. "3" + 2 → "32"). Sicher rechnen: Number("3") + 2.

6. Booleans & Operatoren (&&, ||, !)

Booleans sind Wahrheitswerte: true oder false. Sie steuern Bedingungen in if-Abfragen und Schleifen.

Beispiel

let isLoggedIn = true;
                                            let hasTicket = false;
                                            // Nutze die Konsole für Tests

Konsole: Werte ändern und prüfen (F12 → „Konsole“).


Logische Operatoren

UND (&&): ergibt nur true, wenn beide Operanden true sind. 

console.log(true && true);   // true
                                        console.log(true && false);  // false
                                        console.log(false && false); // false

ODER (||): ergibt true, wenn mindestens einer true ist. 

console.log(true || false);  // true
                                        console.log(false || false); // false
                                        console.log(true || true);   // true

NICHT (!): kehrt den Wert um.

console.log(!true);  // false
                                        console.log(!false); // true

Konsole: mit eigenen Kombinationen spielen, z. B. (a && b) || c.


Alltagsbeispiel

let alter = 20;
        let ticket = true;
        let isVIP = false;
        
        // UND: beide Bedingungen müssen stimmen
        if (alter >= 18 && ticket) {
          console.log("Du darfst rein!");
        }
        
        // ODER: eine Bedingung reicht
        if (alter >= 18 || isVIP) {
          console.log("Du kommst trotzdem rein.");
        }

Konsole: Werte von alter, ticket, isVIP ändern und Ergebnis beobachten.


Vergleichsoperatoren

Vergleiche prüfen zwei Werte und liefern immer einen Boolean.

Gleichheit

console.log(5 == "5");  // true  (nur Wert)
        console.log(5 === "5"); // false (Wert & Typ)
        console.log(5 === 5);   // true

Ungleich

console.log(5 != "5");  // false
        console.log(5 !== "5"); // true

Größer/Kleiner

console.log(10 > 5);   // true
        console.log(5 < 5);    // false
        console.log(5 <= 5);   // true

Konsole: === statt == bevorzugen; sicherstellen, dass Zahlen wirklich Number sind.


Merksatz

Boolean = wahr/falsch · && = UND (beide) · || = ODER (mindestens eins) · ! = NICHT (umkehren) · === vergleicht Wert und Typ.

Konsole: eigene Mini-Checks notieren und ausführen.

Bedingungen (if, else, else if)

- Was ist eine Bedingung?
Mit if prüfst du, ob etwas wahr (true) oder falsch (false) ist.
Nur wenndie Bedingung wahr ist, läuft der Block in geschweiften Klammern.

if – Beispiel

let alter = 20;

if (alter >= 18) {
  console.log("Du bist volljährig.");
}

Ausgabe: Du bist volljährig.


else – wenn die Bedingung nicht erfüllt ist, wird der alternative Block ausgeführt.

let alter = 15;

if (alter >= 18) {
  console.log("Du bist volljährig.");
} else {
  console.log("Du bist noch nicht volljährig.");
}

Ausgabe: Du bist noch nicht volljährig.


else if – mehrere Bedingungen hintereinander prüfen. Der erste passende Block wird ausgeführt.

let punkte = 75;

if (punkte >= 90) {
  console.log("Note: Sehr gut");
} else if (punkte >= 75) {
  console.log("Note: Gut");
} else if (punkte >= 50) {
  console.log("Note: Befriedigend");
} else {
  console.log("Note: Nicht bestanden");
}

Ausgabe: Note: Gut


Ablauf im Überblick: 1) if prüft zuerst. 2) else if prüft weiter, falls zuvor falsch. 3) else greift, wenn nichts davon passt.

Best Practices: klare Bedingungen schreiben (zum Beispiel ===, >=, <=), Reihenfolge von streng zu allgemein wählen, und bei sehr vielen Fällen lieber switch oder eigene Funktionen nutzen.

Merksatz: if bedeutet „wenn“, else bedeutet „ansonsten“, else if bedeutet „wenn nicht das, dann prüfe das“.

Konsole testen: öffne die DevTools (F12), wechsle in die Konsole, füge die Beispiele ein und probiere eigene Werte.

Functions (Funktionen)

- Was ist eine Funktion?
Eine Funktion ist ein wiederverwendbarer Block aus Anweisungen. Sie kann Parameter annehmen und optional ein Ergebnis zurückgeben (return).

Funktions-Deklaration (klassisch)

function greet(name) {
  return "Hallo " + name + "!";
}

let text = greet("Mina");
console.log(text);
Ausgabe: Hallo Mina!


Parameter und Rückgabewert

function add(a, b) {
  return a + b;
}

const summe = add(4, 6);
console.log(summe);
Ausgabe: 10


Funktion ohne Rückgabewert (Seiteneffekt)

function printSum(a, b) {
  console.log("Summe:", a + b);
}

printSum(2, 3);
Ausgabe: Summe: 5


Mehrere Parameter und Standardwerte

function greetLang(name, lang = "de") {
  if (lang === "de") { return "Hallo " + name; }
  if (lang === "en") { return "Hello " + name; }
  return "Hi " + name;
}

console.log(greetLang("Mina"));
console.log(greetLang("Mina", "en"));


Arrow Function (Kurzform)

const double = (x) => x * 2;
console.log(double(7));
Ausgabe: 14


Best Practices

- Eine Funktion sollte genau eine Sache gut erledigen.
- Sinnvolle Namen verwenden (Verb + Objekt, z. B. getUserName, calculateTotal).
- Wenige Parameter bevorzugen; bei vielen lieber ein Objekt übergeben.
- return früh nutzen, um Fälle klar zu beenden.

Merksatz: Funktion kapselt Arbeit, bekommt Eingaben, macht etwas, gibt optional ein Ergebnis zurück.

Konsole testen: DevTools öffnen (F12), in die Konsole wechseln, die Beispiele einfügen und eigene Varianten ausprobieren.

Functions – Parameter (Einführung)

- Parameter sind Platzhalter in einer Funktion.
- Beim Aufruf werden Parameter durch Argumente ersetzt.
- So kannst du Werte von außen in die Funktion geben.


Einfaches Beispiel (wie bei w3schools: kurz, direkt, mit Ergebnis)

function begruessen(name) {
  console.log("Hallo " + name + "!");
}

begruessen("Emil");
begruessen("Nam");

Ausgabe:
Hallo Emil!
Hallo Nam!

- name ist der Parameter.
- Emil und Nam sind die Argumente (konkrete Werte beim Aufruf).


Mehrere Parameter

function addiere(a, b) {
  console.log(a + b);
}

addiere(3, 5);
addiere(10, 20);

Ausgabe:
8
30


Parameter und Rückgabe (return)

function multipliziere(x, y) {
  return x * y;
}

let ergebnis = multipliziere(4, 7);
console.log(ergebnis);

Ausgabe:
28


Best Practices

- Parameter klar benennen (z. B. name, preis, anzahl).
- Vor dem Schreiben überlegen: soll die Funktion etwas ausgeben (console.log) oder etwas zurückgeben (return).
- Funktionen mit Parametern machen Code flexibler und wiederverwendbar.

Merksatz: Parameter sind die Tür zur Funktion — du gibst Werte hinein, die innen verarbeitet werden.

Konsole testen: DevTools öffnen (F12), in die Konsole wechseln, die Beispiele einfügen und eigene Werte probieren.

Objects (Einführung)

- Ein Object ist ein Datentyp in JavaScript, der Daten als Schlüssel-Wert-Paare speichert.
- Damit kannst du komplexe Informationen strukturiert ablegen.
- Beispiel: Eine Person mit Name, Alter und Größe.


Beispiel

let person = {
  name: "Emil",
  alter: 28,
  groesse: 177
};

console.log(person);

- Keys (Schlüssel): name, alter, groesse
- Values (Werte): "Emil", 28, 177


Zugriff auf Werte

console.log(person.name); // Punkt-Notation → Emil
console.log(person["alter"]); // Klammer-Notation → 28


Werte ändern

person.alter = 29; // Wert ändern
person.hobby = "Kochen"; // Neues Feld hinzufügen

console.log(person);
// { name: "Emil", alter: 29, groesse: 177, hobby: "Kochen" }


Warum Objects?

- Zusammenhängende Daten sauber bündeln (z. B. Person, Rezept, Auto).
- Bessere Struktur statt viele einzelne Variablen.
- Grundlage für moderne Datenformate wie JSON, für APIs und Datenbanken.


Merksatz

Eine Variable speichert einen Wert.
Ein Object speichert viele Werte – geordnet nach Schlüssel und Wert.

Strings in JavaScript

- Strings sind Texte, zum Beispiel "Hallo Welt".
- JavaScript bietet viele eingebaute Möglichkeiten, Strings zu untersuchen oder zu verändern.
- Strings verhalten sich wie Objekte mit Eigenschaften und Methoden.


length – Zeichen zählen

- Gibt die Anzahl der Zeichen im String zurück (inklusive Leerzeichen).
- length ist eine Eigenschaft, keine Funktion.

let text = "Hallo Welt";
console.log(text.length);

Ausgabe: 10


trim() – Leerzeichen an den Rändern entfernen

- Entfernt Leerzeichen am Anfang und am Ende eines Strings.
- Praktisch bei Nutzereingaben (Formulare).
- trim() verändert den ursprünglichen String nicht, sondern liefert eine neue Version.

let eingabe = " Emil ";
console.log(eingabe);
console.log(eingabe.trim());

Ausgabe:
  " Emil "
  "Emil"


Praxisbeispiel

let userInput = " test@mail.com ";

if (userInput.trim().length > 0) {
  console.log("Eingabe gültig: " + userInput.trim());
} else {
  console.log("Feld darf nicht leer sein!");
}


Merksatz

- length zählt die Zeichen im String.
- trim() schneidet überflüssige Leerzeichen am Anfang und Ende ab.

Konsole testen: DevTools öffnen (F12), die Beispiele einfügen und mit eigenen Werten ausprobieren.

Strings – Überblick

- Strings sind Texte, zum Beispiel "Hallo Welt".
- Sie besitzen eingebaute Eigenschaften und Funktionen, um mit Text zu arbeiten.
- Du kannst zählen, prüfen, umwandeln, suchen, ersetzen und zuschneiden.

W3 Schools

Eigenschaft: length

- Gibt die Anzahl der Zeichen im String zurück (inklusive Leerzeichen).
- length ist eine Eigenschaft, keine Funktion.

let wort = "Kochwelt";
console.log(wort.length);
Ausgabe: 8


trim() – Leerzeichen an den Rändern entfernen

- Entfernt Leerzeichen am Anfang und am Ende eines Strings.
- Praktisch für Nutzereingaben (Formulare).
- Verändert nicht den Original-String, sondern liefert eine neue Version.

let eingabe = " Emil ";
console.log(eingabe);
console.log(eingabe.trim());
Ausgabe:
  " Emil "
  "Emil"


toUpperCase() / toLowerCase() – Groß/Klein

- Wandelt den ganzen Text in Groß- bzw. Kleinbuchstaben.

console.log("hallo".toUpperCase());
console.log("WELT".toLowerCase());
Ausgabe:
  "HALLO"
  "welt"


includes() – Teilstring enthalten?

- Prüft, ob ein bestimmter Text vorkommt. Liefert true oder false.

console.log("Kochwelt".includes("welt"));
Ausgabe: true


startsWith() / endsWith() – beginnt/endet mit?

- Prüft den Anfang bzw. das Ende eines Strings.

console.log("JavaScript".startsWith("Java"));
console.log("JavaScript".endsWith("Script"));
Ausgabe:
  true
  true


replace() – Teil ersetzen

- Ersetzt den ersten passenden Teil durch neuen Text.

console.log("Hallo Welt".replace("Welt", "Emil"));
Ausgabe: "Hallo Emil"


slice() / substring() – Teil ausschneiden

- Schneidet Abschnitte aus einem String heraus (Startindex, optional Endindex exklusiv).

console.log("Kochwelt".slice(0, 4));
Ausgabe: "Koch"


Wichtiges zu Strings

- Strings sind unveränderlich (immutable). Funktionen liefern neue Strings zurück.
- Methoden lassen sich gut kombinieren (Method Chaining).

let email = " TEST@MAIL.COM ";
console.log(email.trim().toLowerCase());
Ausgabe: "test@mail.com"


Merksatz

- length zählt Zeichen.
- trim schneidet Ränder.
- toUpperCase/toLowerCase ändern das Format.
- includes, startsWith, endsWith prüfen Inhalte.
- replace ersetzt.
- slice/substring schneiden Stücke heraus.

Konsole testen: DevTools öffnen (F12), Beispiele einfügen und mit eigenen Texten experimentieren.
